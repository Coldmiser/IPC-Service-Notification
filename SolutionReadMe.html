Service

The service is written as a Console application using the TopShelf plug-in so troubleshooting is easier. It can be run as a Console application, but is designed as a service. Information on the TopShelf plug-in can be found here

The service can be installed by running the application with the following command line parameters:

  • service.exe install start
  • service.exe uninstall
  • service.exe -instance (if registering the service multiple times)
  • service.exe -username -password
  • service.exe help

Deployment

The deployment method used for this solution is the Visual Studio Package Manager. I had previously used Squirrel Package Manager with it, but it did not give me the options that I was looking for so I am currently switching off of Squirrel and onto the VS Package Manager.

The package is deployed by using Squirrel Package manager, this is done from with-in the Visual Studio Package Manager Console. Simply select the Package Manager Console (View → Other Windows → Package Manager Console) and type: Squirrel --releasify Service.1.0.0.nupkg. This will use your nupkg file to create the EXE and MSI files. A new nupkg file should be created for each release and added to source control.
